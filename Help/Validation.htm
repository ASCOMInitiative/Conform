Introduction

Conform provides the capability for driver authors to create a validation file that others can use to confirm that the supplied driver does achieve a specified level of conformance to the relevant ASCOM specification.

Creating a validation file

Creation of a validation file is effected by clicking on the "Create Validation File" entry in the "Validation" menu. This does not of itself create the validation file, it just flags that you want a validation file created each time you do a conformance check.

Assuming the flag is now set, a conformance check will run completely as normal except for some extra steps at the end that:

  • Create a hash checksum for the driver executable that you are testing
  • Create the validation file that contains the conformance output including the executable checksum
  • Creates a checksum for the whole validation file, including the executable checksum
  • Prepends the validation file checksum to the validation file
  • Reports where the validation file has been created

The executable can now be packaged into an installer, if this has not already been done, and sent together with the validation text file to Bob for inclusion on the driver download page.

Confirming Validation

To confirm validation:

  • Obtain the driver installer and the validation text file from the ASCOM site
  • Install the driver
  • Start Conform and select the driver as if you were going to check its conformance (you do not need to actually run a conformance check)
  • Select "Confirm Current Driver Validation" from the "Validate" menu
  • Select the validation text file through the open file dialogue and press the "Open" button

Conform will then confirm that:

  • the contents of the validation file have not changed from the point at which the driver author created it
  • the installed driver name is the same as in the validation file
  • the installed executable has the same hash checksum as in the validation file

and report accordingly.

Notes

  1. The validation approach validates the executable not the installer
  2. Validation does not force any level of compliance on the driver author. Conform will create a validation file for a driver regarless of how many errors, warnings or issues it generates.
  3. By validating the driver executable rather than an installer, the validation text file only needs to be updated when a driver code change is made.
